Defining controller

Example controller: dynamic/dada/CornerNormalSkullTrumpet.html.aos.xml

Defines a URI: /dada/CornerNormalSkullTrumpet.html

<?xml version="1.0" encoding="utf-8"?> 
<controller>

<!-- This module configures the dada template processor -->
  <module class="Dada.Properties">
    <dataset>
      <default>dada</default>
      <query-parameter>dataset</query-parameter>
    </dataset>
  </module>

<!--
aos:dada template node handler was added to the templates

The template will execute and process appropriate nodes
See Template output generator documentation 
 -->
  <output class='Template'>
    <filename>dada/template_ddt/DDT_CornerNormalSkullTrumpet.template.html</filename>
  </output>

</controller>

Data.Template parameters

dataset/query-parameter - query parameter name to check and if present use value as the dataset name

dataset/default - default dataset to use if query parameter does not override it

Adding dada template section

<aos:dada>content to be processed goes here</aos:dada>

Defining data set of words

Configuring the dataset in conf/AOS_DadaData.xml

<dataset name="dada">

<!-- Set specific data -->

<data type="noun">dada/dataset_dada/noun.dat</data>

<data type="noun_proper">dada/dataset_dada/noun_proper.dat</data>

<data type="verb">dada/dataset_dada/verb.dat</data>

<data type="adjective">dada/dataset_dada/adjective.dat</data>

<data type="adverb">dada/dataset_dada/adverb.dat</data>

<data type="location">dada/dataset_dada/location.dat</data>

<!-- Inherit from common base data -->

<data type="html">dada/base/html.dat</data>

<data type="connect">dada/base/connect.dat</data>

<data type="preposition">dada/base/preposition.dat</data>

<data type="pronoun_s">dada/base/pronoun_singular.dat</data>

<data type="pronoun_p">dada/base/pronoun_plural.dat</data>

<data type="pronoun_s_pos">dada/base/pronoun_singular_possesive.dat</data>

<data type="pronoun_p_pos">dada/base/pronoun_plural_possesive.dat</data>

<data type="pronoun_s_pos_pers">dada/base/pronoun_singular_possesive_personal.dat</data>

<data type="pronoun_p_pos_pers">dada/base/pronoun_plural_possesive_personal.dat</data>

</dataset>

The type defined for each data element is the name that will be used to refer to it using %TYPE. Path is relative to _webroot/data directory.

Word replacement engine

Word Type Replacement

{%TYPE:TAG,TAG...}

TAG applicable to all TYPEs:

article - Prepends a or an to the word

plural - will try to guess the best way to make the word plural

uppercase - will convert the entire word to upper case

lowercase - will convert the entire word to lower case

proper - will capitalize the first letter of the word

TAG applicable to a TYPE called verb:

present - appends -ing ending to the verb

past - append -ed or -d to the verb

TAG to add the word to the variable map

{$KEY:TAG,TAG...}

inserts the current word into the map as myname -> current word

this can be accessed using {$myname} (see below)

TAG applicable to variable insertion:

article - Prepends a or an to the word

plural - will try to guess the best way to make the word plural

uppercase - will convert the entire word to upper case

lowercase - will convert the entire word to lower case

proper - will capitalize the first letter of the word

Adding attributes to the entire line

{&NAME:VALUE}

adds line element attribte for the given line

Input:

{&type:title}Line contents here!

Output:

<line type="title">Line contents here!<line>
