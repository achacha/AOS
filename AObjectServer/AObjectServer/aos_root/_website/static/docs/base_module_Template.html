MODULE: Template

Processes a template and inserts output into the model.
LuaScript step only processes a Lua script while a template may contains Lua, plain text and other template tokens.

USAGE
<module class="Template">
  <template><![CDADA[
... template goes here ...
  ]]></template>
  <path>template/output</path>
</module>

Execute inline script


<module class="Template">
  <filename>script/something.html</filename>
  <path>template/output</path>
</module>

Use a script file


template = actual template
filename = file containing a template to execute located relative to aos_root/data directory
path = destination of the output and is relative to the model root or absolute starting with /root/

EXAMPLES
<module class="Template">
  <template>
<![CDADA[

Plain text,

print "Lua script";
model.addText("/root/justinserted","model text");
, /root/justinserted!

]]></template>
  <path>/root/myoutput</path>
</module>
Results in /root/myoutput element to contain text: 
Plain text, Lua script, model text!

This simple template inlines Some text, Lua script which also inserts into the model and model lookup.
