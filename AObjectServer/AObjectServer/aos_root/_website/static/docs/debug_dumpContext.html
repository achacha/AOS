Debugging with dumpContext query string parameter

Lets assume that you are trying to get the response to http://localhost/ping?dumpContext=1 (or 2)


Assuming your AObjectServer.xml has /server/debug/allow-dumpContext set to true (this should be false in production), you can add this as a parameter to any URL.

dumpContext=1 adds AOSContext object as XML to the model as /root/dumpContext and switches the output generation to XML.



dumpContext=2 more extensive dump that converts buffers to ascii-hex display for more fine-grained analysis, the output tends to be quite large as a result.
