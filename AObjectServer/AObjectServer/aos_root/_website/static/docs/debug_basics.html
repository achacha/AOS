Debugging a request

Lets assume that you are trying to get the response to http://localhost/mypage.xml and it is returning error 500 or not returning what you expected.

Some steps you can take to locate the problem

  1. View the XML using dumpContext parameter
  2. Connect to the admin console and check
    AOSContextManager
    (may need to set debug level to 5 and retry the request with debug level logging, errors are logged as level 1, so look for 1: in the event log)
  3. Add a breakpoint in your module and make sure the code works as expected


Common gotchas

  • Forgetting trailing / at the end of a directory name, since you can have both http://localhost/foo and http://localhost/foo/ the server does not automatically assume that a name is really a directory. In the case of trailing / then default filename is automatically appended otherwise it is literal in path translation.
