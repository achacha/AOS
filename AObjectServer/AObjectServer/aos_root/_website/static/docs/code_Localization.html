Localization


RESOURCE token

On startup all files in aos_root/conf/resources/ are lodead, filenames are locale.xml (e.g. en-us.xml, ru-ru.xml, etc). For a given locale (determined by Session override or Accept-Languages: HTTP header) the resource is looked up and content is emitted.


xml_path

Example
Assuming HTTP request has:
Accept-Languages: en-pirate

and

aos_root/conf/resources/en-pirate.xml is as follows:

<resource> <message>Arrr!</message> </resource>
then the template token evaluates to:

/resource/message = Arrr!

If you provide a locale specific XML resource collection for each language then when it changes the correct one will be used when template is output. This is in addition to the locale specific content provided.
