OUTPUT: From template file(s)

Use a template file or a set of template files to render output.
if and ifnot attributes allow inclusion of templates based on presence/absence of elements as tests. The templates are all streamed out to output in the order they are listed.
You can have both if and ifnot on a template

USAGE
<module class="Template"> <mime-extension>html</mime-extension> <filename if="/root/somepath/exists">script/something.html</filename> <template if="/root/somepath/exists">/root/somepath/exists = <aos:model>/root/somepath/exists</aos:model></template> <template>/root/executed = <aos:model>/root/executed</aos:model></template> <filename ifnot="/root/somepath/doesnotexist">script/something_not.html</filename> <template>Some template text inserted here</template> <filename if="/root/somepath/alsoexists">script/something_more.html</filename> </module> 
filename: template filename
  if: checks for existence of model element
  ifnot: check for absence of model element

template: inlined template code
  if: checks for existence of model element
  ifnot: check for absence of model element

mime-extension: (optional) sets the MIME content type from extension, if not present, the extension of the request URL is used (and if there is non, XML is the default), this is NOT the actual MIME type but rather an extension to lookup in mime_types.ini

OUTPUT
Sequence of templates concatinated together and if they meet conditions (if specified)

EXAMPLES
<output class="Template"> <mime-extension>html</mime-extension> <filename if="SESSION/data/user/isLoggedIn">template/logout.html</filename> <filename ifnot="SESSION/data/user/isLoggedIn">template/login.html</filename> </output>
Display correct template depending on element presence
