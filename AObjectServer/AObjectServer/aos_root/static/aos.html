NOTE: .html extension is remapped to a dynamic command, these are no longer static pages once dynamic command is defined
If you turn off the dynamic command and there exists a static page with the same name, then dynamic command will not execute and static page will load instead. This allows you to have fallbacks when developing dynamic content, you can develop the static page first then "replace" it with a dynamic one and if needed you can turn off the dynamic and start using the static page again. Can be very useful under high loads conditions.
